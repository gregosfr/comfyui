🗺️ Guide d'utilisation Qwen dans ComfyUI

🖼 Génération image (T2I)
  • → Modèle : Qwen-Image-2512
  • → Node : Load Diffusion Model
  • → TE : qwen_2.5_vl_7b_fp8_scaled
  • → VAE : qwen_image_vae.safetensors
  • → CFG : 3.5–5.0 | Steps : 20–30
  • → Résolution : 1024–1328px
  • → LoRA Lightning = 4 steps
✏️ Édition image
  • → Modèle : Qwen-Image-Edit-2511
  • → Nunchaku INT4 disponible
  • → Même VAE + TE que 2512
  • → Fonctionne en inpainting
  • → Guidage textuel précis
  • → Préférer FP8 ou INT4
💬 Améliorer ses prompts
  • → Modèle : Qwen3-VL 4B ou 7B
  • → Plugin : ComfyUI-QwenVL
  • → Entrée : prompt court
  • → Sortie : prompt enrichi détaillé
  • → Peut analyser une image source
  • → Fonctionne avec tous les modèles

🏁

Flags VRAM essentiels

# RTX 3060 6GB

🧠 Techniques avancées

🎯

TeaCache

Double la vitesse de génération FLUX avec une légère perte de qualité.

Valeur : 0.25–0.30
Nœud : ComfyUI-TeaCache
×2 vitesse FLUX

GGUF Quantization

Réduit drastiquement la VRAM nécessaire avec peu de perte de qualité.

Q4_K_M : -50% VRAM, -5% qualité
Q5_K_M : -40% VRAM, -2% qualité
Q6_K : -30% VRAM, ≈ identique
Q8_0 : -15% VRAM, ≈ fp16
→ Plugin ComfyUI-GGUF
💾

CPU Offloading CLIP/T5

Charger les text encoders en RAM CPU, libère la VRAM pour le UNet.

# DualCLIPLoader → device: cpu
T5-XXL Q8 en RAM = 5 GB RAM
VRAM libérée pour UNet
Surcoût : ~5–10% de temps
Recommandé RTX 3060
🔢

CFG Scale optimal

Un CFG mal réglé détruit la qualité et gaspille du temps.

FLUX : 1.0–3.5 (très bas !)
SD 1.5 : 7–12
SDXL : 5–9
SD3.5 : 4.5–7
Vidéo Wan/Hunyuan : 6–7
🎲

Sampler & Steps optimaux

Le bon sampler économise des steps sans perdre en qualité.

FLUX : Euler, 15–20 steps
SDXL : DPM++ 2M Karras, 25–30
SD3.5 : DPM++ 2M, 20–28
Distillé (Schnell/Turbo) : Euler, 4–8
Plateau qualité : 30 steps max
🖼

Upscale en 2 passes (Hires Fix)

Générez en petite résolution + upscale = moins de VRAM, meilleure qualité.

1️⃣ Génération : 512–768px
2️⃣ Upscale x2 : RealESRGAN
3️⃣ img2img : denoise 0.4–0.6
Résultat : 1024–2048px net
-40% VRAM vs direct

  • • Surveiller avec GPU-Z ou HWiNFO64

Drivers NVIDIA

  • • CUDA 12.x requis pour les modèles 2025
  • • PyTorch 2.x recommandé
  • • Python 3.10 ou 3.11
🧹

Libérer la VRAM

  • • Fermer Chrome / Discord (accél. GPU)

  • • Node "Free GPU Memory" en fin de workflow
  • • Redémarrer ComfyUI entre gros modèles
